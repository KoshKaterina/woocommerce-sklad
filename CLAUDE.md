# CLAUDE.md — ориентир по проекту

Интеграция заказов **WooCommerce / InSales / uCoz → МойСклад (МС)**. Односторонняя:
заказ создаётся в источнике → создаётся «Заказ покупателя» в МС. Поля заказа после
создания не перезаписываются (менеджеры правят вручную), кроме reverse-sync (см. ниже).

Python 3.12 / FastAPI / structlog. Деплой — Docker (`Dockerfile` копирует `woo_moysklad/`,
запускает `uvicorn woo_moysklad.main:app`).

## Команды
- Тесты: `.venv/bin/python3 -m pytest tests/ -q` (сейчас ~193, должны быть зелёными).
- Ручной прогон заказа: `python scripts/process_order.py <id>` (WC),
  `python scripts/process_insales_order.py <номер>` (InSales),
  `python scripts/resync_order.py --order <N> --dry-run` (reverse-sync 1 заказа).
  Скрипты запускаются из корня репо (внутри есть bootstrap пути — `PYTHONPATH` не нужен).
- InSales API доступен **только с российского IP** (иначе 403 «Доступ ограничен»).

## Архитектура (пакет `woo_moysklad/`)
Все источники нормализуются в общий `NormalizedOrder`, дальше единый `OrderProcessor`.
- `main.py` — FastAPI: вебхук `POST /webhook/order` (WC, HMAC), `/health`, lifespan
  (запускает сверку и, опционально, uCoz-поллер). Точка входа.
- `config.py` — конфиг из `.env` + **захардкоженные дефолты** (`_HARDCODED_DEFAULTS`).
- `logger.py`, `exceptions.py`, `ms_client.py` (HTTP-клиент МС, общий).
- `core/` — платформо-независимый конвейер:
  `normalized_order`, `order_processor`, `counterparty_handler`, `product_matcher`,
  `field_mappers`, `reconciliation`, `source_adapter`, `field_resync`.
- `woocommerce/` (`client`, `normalizer`), `insales/` (`client`, `normalizer`),
  `ucoz/` (`client`, `normalizer`, `poller`, `state`).
- `scripts/` — ручные/разовые утилиты (НЕ в Docker-образе).

## Источники — статус
- **WooCommerce** — в проде. Вебхук + сверка. Канал «Магазин», без суффикса номера.
- **InSales** — готов, через сверку (вебхука нет). Включается, если в `.env` заданы
  `INSALES_API_KEY` + `INSALES_PASSWORD` (остальное захардкожено). Канал TangemShop,
  организация ИП Абовян, суффикс номера ` Tangemshop`.
- **uCoz (TG-магазин)** — **WIP, НЕ закоммичен** (пакет `woo_moysklad/ucoz/` untracked).
  `main.py` импортирует его **лениво** (только при заданном `UCOZ_POLL_URL`). В прод не включать.

## Сверка (`core/reconciliation.py`) — раз в 3 мин, окно 9 мин
Берёт заказы, **изменённые** за окно (WC `date_modified`, InSales `updated_at`), и:
заказа нет в МС → создать; есть, надо оплатить, `payedSum==0` → проставить платёж.
Поля существующих заказов НЕ трогает. Поиск в МС — по доп.полю «Номер заказа на сайте».

## Reverse-sync (`core/field_resync.py`) — `FIELD_RESYNC_ENABLED`, по умолч. ВКЛ
Идёт по заказам МС, изменённым в окне (кроме канала «Маркетплейс»), и пересчитывает
зависимые доп.поля из позиций + «Способ оплаты»: Оценочная = Σ товаров, Стоимость
доставки = Σ услуг, Итого = Σ всего при COD / 0 при предоплате, Прием платежа.
Пишет **только дифф** (идемпотентно, без зацикливания). Не трогает контрагента/статус/
товары (кроме обнуления цены услуги СДЭК при «на карту»/«банковский перевод»).

## Ключевые доменные правила
- **Доп.поля МС (2026-06):** «Вид доставки» — `long` (1=ПВЗ,2=курьер,3=почтомат),
  стоимости — `double`, «Прием платежа» — справочник (1=предоплата, 2=безнал картой,
  3=наличные). UUID **захардкожены** в `config.py`, не из `.env`.
- **Оплата (WC):** «на карту»/«банковский перевод» → предоплата + бесплатная СДЭК-доставка
  + не помечать оплаченным авто (`is_manual_prepayment`); «онлайн» → предоплата без обнуления;
  «при получении» → COD.
- **Контрагент:** ищется по телефону; если у найденного имя-заглушка (телефон) или пустой
  email — дозаполняем из заказа. Имя при создании InSales берётся из shipping_address,
  если в client.name телефон.

## Конфиг и `.env`
- В коде захардкожены (`_HARDCODED_DEFAULTS`, env их НЕ перебивает): UUID мигрированных
  доп.полей, каналы (TangemShop/Маркетплейс), параметры InSales-источника (URL, организация,
  статус, проект). Менять — только в коде.
- В `.env` — секреты и WC/МС-идентификаторы: `MS_TOKEN`, `WC_*`, `MS_ORGANIZATION_ID`,
  `MS_STORE_*`, неизменённые `MS_ATTR_*`, и для InSales — только `INSALES_API_KEY`/`PASSWORD`.
- `FIELD_RESYNC_ENABLED` — bool, дефолт `true`; отключить — `=false` в `.env`.
- `.env` гитом не отслеживается — прод-`.env` правится вручную на сервере.
